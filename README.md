# Chest X-Ray Pneumonia Classifier

A web app that takes a chest X-ray image and predicts whether it shows signs of **Pneumonia** or is **Normal**, using a fine-tuned EfficientNetB3 model served through a FastAPI backend.

## Overview

Three models were trained and compared for this task:

| Model | Description |
|---|---|
| Scratch CNN | Baseline CNN built from scratch |
| DenseNet121 | Transfer learning baseline |
| **EfficientNetB3** | **Best performing model — used in this app** |

The final model takes a chest X-ray image as input and outputs a single probability:
- **0 → NORMAL**
- **1 → PNEUMONIA**

## Tech Stack

- **Model:** TensorFlow / Keras (EfficientNetB3, fine-tuned)
- **Backend:** FastAPI
- **Frontend:** HTML, CSS, JavaScript (vanilla)
- **Server:** Uvicorn

## Dataset

The model was trained on the **Chest X-Ray Images (Pneumonia)** dataset:

🔗 [Kaggle: Chest X-Ray Images (Pneumonia)](https://www.kaggle.com/datasets/paultimothymooney/chest-xray-pneumonia)

This dataset contains labeled chest X-ray images split into `NORMAL` and `PNEUMONIA` classes, originally sourced from pediatric patients at Guangzhou Women and Children's Medical Center.

> Note: this Kaggle link is the **training dataset**, not the trained model. If you want to retrain from scratch, download this dataset. If you just want to *run the app*, you need the trained `.keras` weights file (see below).

## Getting the Trained Model

The trained `.keras` model file is **not included in this repository** (model weight files are large and not suited for git version control).

To run this app:
1. Get the trained model file (`m1-2.keras`) — contact the repo owner or retrain using the notebook in `notebooks/` with the Kaggle dataset above.
2. Place it inside the `models/` folder so the path matches what's expected in `main.py`:
   ```
   models/m1-2.keras
   ```

## Setup & Installation

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/pneumonia-xray-classifier.git
cd pneumonia-xray-classifier
```

### 2. Create and activate a virtual environment

```bash
python -m venv .venv
```

**Windows (PowerShell):**
```powershell
.venv\Scripts\activate
```

**macOS/Linux:**
```bash
source .venv/bin/activate
```

### 3. Install dependencies

For the **container/app runtime**, install only the minimal set:

```bash
pip install -r requirements.txt
```

For **Jupyter notebook / training / experimentation** work, install the broader dev set instead:

```bash
pip install -r requirements-dev.txt
```

### 4. Add the trained model

Place your `.keras` model file into the `models/` folder (see [Getting the Trained Model](#getting-the-trained-model) above).

### 5. Run the app

```bash
uvicorn main:app --reload
```

### 6. Open in browser

```
http://127.0.0.1:8000
```

Upload a chest X-ray image and click **Predict** to get a result.

## API Reference

### `GET /`
Returns the upload page (HTML).

### `POST /predict`
Accepts a multipart form upload with a single image file.

**Request:**
- `file`: image file (jpg/png)

**Response:**
```json
{
  "prediction": "PNEUMONIA",
  "confidence": 0.91
}
```

Interactive API docs are auto-generated by FastAPI at:
```
http://127.0.0.1:8000/docs
```
